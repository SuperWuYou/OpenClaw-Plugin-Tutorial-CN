# 第三章：核心概念

本章将深入介绍 OpenClaw 插件系统的核心概念和架构原理。理解这些概念对于开发高质量的插件至关重要。

## 3.1 插件生命周期

插件从发现到运行经历以下阶段：

```
┌─────────────────────────────────────────────────────────────┐
│                       插件生命周期                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 发现 (Discovery)                                        │
│     └─ 扫描扩展目录，查找 openclaw.plugin.json              │
│                          ↓                                  │
│  2. 验证 (Validation)                                       │
│     └─ 验证清单文件和配置 Schema                            │
│                          ↓                                  │
│  3. 加载 (Loading)                                          │
│     └─ 使用 jiti 加载 TypeScript 模块                       │
│                          ↓                                  │
│  4. 注册 (Registration)                                     │
│     └─ 调用 register() 函数，注册功能                       │
│                          ↓                                  │
│  5. 运行 (Running)                                          │
│     └─ 插件功能处于活跃状态                                  │
│                          ↓                                  │
│  6. 停止 (Shutdown)                                         │
│     └─ 调用服务的 stop() 方法（如果有）                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.1.1 发现阶段

OpenClaw 按以下优先级扫描插件：

| 优先级 | 位置 | 说明 |
|--------|------|------|
| 1 (最高) | `plugins.load.paths` | 配置文件中指定的路径 |
| 2 | `<workspace>/.openclaw/extensions/` | 工作区本地插件 |
| 3 | `~/.openclaw/extensions/` | 用户全局插件 |
| 4 (最低) | `<openclaw>/extensions/` | 内置插件（默认禁用） |

### 3.1.2 验证阶段

验证过程检查：

- `openclaw.plugin.json` 是否存在
- `id` 字段是否有效
- `configSchema` 是否符合 JSON Schema 规范
- 安全检查（路径遍历、权限等）

### 3.1.3 注册阶段

```typescript
// 插件注册流程
const plugin = {
  id: "my-plugin",
  register(api: OpenClawPluginApi) {
    // 1. 注册工具
    api.registerTool({ /* ... */ });

    // 2. 注册钩子
    api.registerHook("message_received", handler);

    // 3. 注册命令
    api.registerCommand({ /* ... */ });

    // 4. 注册服务
    api.registerService({ /* ... */ });
  },
};
```

## 3.2 插件 API (OpenClawPluginApi)

`OpenClawPluginApi` 是插件与 OpenClaw 核心交互的主要接口。

### 3.2.1 API 属性

```typescript
interface OpenClawPluginApi {
  // ========== 基本信息 ==========
  /** 插件 ID */
  id: string;

  /** 插件名称 */
  name: string;

  /** 插件版本 */
  version?: string;

  /** 插件描述 */
  description?: string;

  /** 插件来源路径 */
  source: string;

  // ========== 配置访问 ==========
  /** OpenClaw 全局配置 */
  config: OpenClawConfig;

  /** 当前插件的配置 */
  pluginConfig?: Record<string, unknown>;

  // ========== 运行时访问 ==========
  /** 运行时 API */
  runtime: PluginRuntime;

  /** 日志记录器 */
  logger: PluginLogger;

  // ========== 路径解析 ==========
  /** 解析相对路径 */
  resolvePath: (input: string) => string;
}
```

### 3.2.2 API 方法

```typescript
interface OpenClawPluginApi {
  // ========== 工具注册 ==========
  /**
   * 注册 Agent 工具
   * @param tool 工具定义或工具工厂函数
   * @param opts 可选配置
   */
  registerTool(
    tool: AnyAgentTool | OpenClawPluginToolFactory,
    opts?: OpenClawPluginToolOptions,
  ): void;

  // ========== 钩子注册 ==========
  /**
   * 注册事件钩子
   * @param events 事件名称或事件数组
   * @param handler 事件处理器
   * @param opts 可选配置
   */
  registerHook(
    events: string | string[],
    handler: InternalHookHandler,
    opts?: OpenClawPluginHookOptions,
  ): void;

  // ========== HTTP 路由 ==========
  /**
   * 注册 HTTP 路由
   * @param params 路由配置
   */
  registerHttpRoute(params: OpenClawPluginHttpRouteParams): void;

  // ========== 渠道注册 ==========
  /**
   * 注册消息渠道
   * @param registration 渠道插件
   */
  registerChannel(
    registration: OpenClawPluginChannelRegistration | ChannelPlugin,
  ): void;

  // ========== Gateway 方法 ==========
  /**
   * 注册 Gateway RPC 方法
   * @param method 方法名称
   * @param handler 处理器
   */
  registerGatewayMethod(
    method: string,
    handler: GatewayRequestHandler,
  ): void;

  // ========== CLI 命令 ==========
  /**
   * 注册 CLI 命令
   * @param registrar 注册函数
   * @param opts 可选配置
   */
  registerCli(
    registrar: OpenClawPluginCliRegistrar,
    opts?: { commands?: string[] },
  ): void;

  // ========== 后台服务 ==========
  /**
   * 注册后台服务
   * @param service 服务定义
   */
  registerService(service: OpenClawPluginService): void;

  // ========== 提供商 ==========
  /**
   * 注册模型提供商
   * @param provider 提供商定义
   */
  registerProvider(provider: ProviderPlugin): void;

  // ========== 自定义命令 ==========
  /**
   * 注册自动回复命令
   * @param command 命令定义
   */
  registerCommand(command: OpenClawPluginCommandDefinition): void;

  // ========== 生命周期钩子 ==========
  /**
   * 注册生命周期钩子（新 API）
   * @param hookName 钩子名称
   * @param handler 处理器
   * @param opts 可选配置
   */
  on<K extends PluginHookName>(
    hookName: K,
    handler: PluginHookHandlerMap[K],
    opts?: { priority?: number },
  ): void;
}
```

## 3.3 插件定义结构

插件可以以两种形式导出：

### 3.3.1 对象形式（推荐）

```typescript
// 定义插件对象
const myPlugin = {
  // ========== 元数据 ==========
  id: "my-plugin",
  name: "My Plugin",
  description: "插件描述",
  version: "1.0.0",
  kind: "memory", // 可选：插件类型

  // ========== 配置 Schema ==========
  configSchema: {
    parse(value: unknown) {
      return value || {};
    },
    jsonSchema: {
      type: "object",
      additionalProperties: false,
      properties: {
        apiKey: { type: "string" },
      },
    },
    uiHints: {
      apiKey: { label: "API Key", sensitive: true },
    },
  },

  // ========== 注册函数 ==========
  register(api: OpenClawPluginApi) {
    // 注册功能...
  },

  // ========== 激活函数（可选）==========
  activate(api: OpenClawPluginApi) {
    // 插件激活时调用...
  },
};

export default myPlugin;
```

### 3.3.2 函数形式（简单场景）

```typescript
// 简单的函数形式导出
export default function register(api: OpenClawPluginApi) {
  // 直接注册功能
  api.registerTool({
    name: "my_tool",
    // ...
  });
}
```

## 3.4 配置系统

### 3.4.1 配置 Schema

插件使用 JSON Schema 定义配置结构：

```typescript
// 完整的配置 Schema 示例
const configSchema = {
  // 解析函数：将原始配置转换为类型化的配置对象
  parse(value: unknown): MyConfig {
    // 处理原始值
    const raw = value && typeof value === "object" ? value : {};

    return {
      enabled: raw.enabled ?? true,
      apiKey: raw.apiKey || process.env.MY_API_KEY,
      timeout: raw.timeout ?? 30000,
    };
  },

  // JSON Schema：用于配置验证和 UI 生成
  jsonSchema: {
    type: "object",
    additionalProperties: false,  // 禁止未知属性
    properties: {
      enabled: {
        type: "boolean",
        default: true,
      },
      apiKey: {
        type: "string",
      },
      timeout: {
        type: "number",
        minimum: 1000,
        maximum: 60000,
      },
      region: {
        type: "string",
        enum: ["us-east-1", "eu-west-1", "ap-northeast-1"],
      },
    },
  },

  // UI 提示：用于 UI 渲染
  uiHints: {
    apiKey: {
      label: "API 密钥",
      sensitive: true,  // 标记为敏感信息
      placeholder: "输入您的 API 密钥",
    },
    timeout: {
      label: "超时时间（毫秒）",
      help: "请求超时时间，默认 30000ms",
    },
    region: {
      label: "服务区域",
      help: "选择离您最近的区域以获得最佳性能",
    },
  },
};
```

### 3.4.2 配置验证规则

| 规则 | 说明 |
|------|------|
| `type` | 指定数据类型 |
| `additionalProperties: false` | 禁止未定义的属性 |
| `required` | 指定必需属性 |
| `default` | 设置默认值 |
| `enum` | 限制可选值 |
| `minimum/maximum` | 数值范围限制 |
| `pattern` | 字符串正则匹配 |

### 3.4.3 UI 提示属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `label` | string | 字段显示标签 |
| `help` | string | 帮助文本 |
| `placeholder` | string | 输入框占位符 |
| `sensitive` | boolean | 是否为敏感信息（如密码） |
| `advanced` | boolean | 是否为高级选项 |
| `tags` | string[] | 标签 |

## 3.5 钩子系统

钩子允许插件在特定事件发生时执行代码。

### 3.5.1 可用钩子事件

```typescript
// 钩子事件类型
type PluginHookName =
  // ========== 模型相关 ==========
  | "before_model_resolve"    // 模型解析前
  | "before_prompt_build"     // 提示词构建前
  | "before_agent_start"      // Agent 启动前
  | "llm_input"               // LLM 请求时
  | "llm_output"              // LLM 响应时
  | "agent_end"               // Agent 结束时

  // ========== 会话相关 ==========
  | "before_compaction"       // 会话压缩前
  | "after_compaction"        // 会话压缩后
  | "before_reset"            // 会话重置前
  | "session_start"           // 会话开始
  | "session_end"             // 会话结束

  // ========== 消息相关 ==========
  | "message_received"        // 收到消息
  | "message_sending"         // 发送消息前
  | "message_sent"            // 消息发送后

  // ========== 工具相关 ==========
  | "before_tool_call"        // 工具调用前
  | "after_tool_call"         // 工具调用后
  | "tool_result_persist"     // 工具结果持久化前
  | "before_message_write"    // 消息写入前

  // ========== 子代理相关 ==========
  | "subagent_spawning"       // 子代理生成中
  | "subagent_delivery_target"// 子代理投递目标
  | "subagent_spawned"        // 子代理已生成
  | "subagent_ended"          // 子代理已结束

  // ========== Gateway 相关 ==========
  | "gateway_start"           // Gateway 启动
  | "gateway_stop";           // Gateway 停止
```

### 3.5.2 注册钩子

```typescript
// 使用 api.on() 注册钩子（推荐）
api.on("message_received", async (event, ctx) => {
  // 处理收到的消息
  console.log(`收到消息: ${event.content}`);
  console.log(`来源: ${event.from}`);
  console.log(`渠道: ${ctx.channelId}`);
});

// 使用 api.registerHook() 注册钩子（旧 API）
api.registerHook(
  ["command:new", "command:reset"],
  async (event) => {
    console.log(`命令触发: ${event.action}`);
  },
  {
    name: "my-plugin.command-handler",
    description: "处理命令事件",
  },
);
```

## 3.6 工具系统

### 3.6.1 工具定义结构

```typescript
// Agent 工具定义
interface AgentTool {
  // 工具名称（snake_case）
  name: string;

  // 显示标签
  label?: string;

  // 工具描述（Agent 根据这个决定是否调用）
  description: string;

  // 参数 Schema（使用 Typebox）
  parameters: TSchema;

  // 执行函数
  execute: (
    toolCallId: string,
    params: Record<string, unknown>,
  ) => Promise<ToolResult>;
}

// 工具结果类型
interface ToolResult {
  content: Array<{
    type: "text" | "image";
    text?: string;
    image_url?: { url: string };
  }>;
  details?: unknown;
}
```

### 3.6.2 工具工厂

工具可以使用工厂函数动态创建：

```typescript
// 工具工厂函数
type OpenClawPluginToolFactory = (
  ctx: OpenClawPluginToolContext,
) => AnyAgentTool | AnyAgentTool[] | null | undefined;

// 上下文包含会话信息
interface OpenClawPluginToolContext {
  config?: OpenClawConfig;
  workspaceDir?: string;
  agentDir?: string;
  agentId?: string;
  sessionKey?: string;
  sessionId?: string;
  messageChannel?: string;
  agentAccountId?: string;
  requesterSenderId?: string;
  senderIsOwner?: boolean;
  sandboxed?: boolean;
}

// 使用示例
api.registerTool(
  (ctx) => {
    // 根据上下文决定是否创建工具
    if (!ctx.workspaceDir) {
      return null;
    }

    return {
      name: "my_tool",
      description: "依赖工作区的工具",
      parameters: Type.Object({
        path: Type.String(),
      }),
      async execute(_toolCallId, params) {
        // 可以访问 ctx 中的信息
        const fullPath = `${ctx.workspaceDir}/${params.path}`;
        // ...
      },
    };
  },
  { optional: true },  // 标记为可选工具
);
```

### 3.6.3 可选工具

```typescript
// 注册可选工具
// 可选工具可以被 Agent 策略禁用
api.registerTool(myTool, { optional: true });

// 注册必需工具（默认）

```
