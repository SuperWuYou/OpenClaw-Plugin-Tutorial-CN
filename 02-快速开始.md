# 第二章：快速开始

本章将引导您创建第一个 OpenClaw 插件。通过一个简单的示例，您将了解插件的基本结构和开发流程。

## 2.1 环境准备

### 2.1.1 系统要求

| 要求 | 说明 |
|------|------|
| Node.js | 版本 18.x 或更高 |
| npm/pnpm | 用于安装依赖 |
| TypeScript | 5.x 或更高（可选，但推荐） |
| 编辑器 | VS Code 或其他支持 TypeScript 的编辑器 |

### 2.1.2 验证环境

```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version

# 检查 OpenClaw 是否安装
openclaw --version
```

## 2.2 创建第一个插件

让我们创建一个简单的 "Hello World" 插件，它将注册一个自定义命令 `/hello`。

### 2.2.1 创建插件目录

```bash
# 进入 OpenClaw 扩展目录
cd ~/.openclaw/extensions

# 创建插件目录
mkdir hello-plugin

# 进入目录
cd hello-plugin
```

### 2.2.2 创建插件清单文件

每个插件必须包含一个 `openclaw.plugin.json` 清单文件：

```json
{
  "id": "hello-plugin",
  "name": "Hello Plugin",
  "description": "一个简单的示例插件，演示基本功能",
  "version": "1.0.0",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 插件唯一标识符，使用小写字母和连字符 |
| `name` | 否 | 插件显示名称 |
| `description` | 否 | 插件描述 |
| `version` | 否 | 语义化版本号 |
| `configSchema` | 是 | JSON Schema，定义配置结构（可以为空） |

### 2.2.3 创建插件入口文件

创建 `index.ts` 文件：

```typescript
// 导入插件 API 类型定义
// 从 openclaw/plugin-sdk/core 导入核心类型
import type { OpenClawPluginApi } from "openclaw/plugin-sdk/core";

// 导入空配置 Schema 工具函数
// 用于不需要配置的插件
import { emptyPluginConfigSchema } from "openclaw/plugin-sdk/core";

// 定义插件对象
const helloPlugin = {
  // 插件 ID，必须与 openclaw.plugin.json 中的 id 一致
  id: "hello-plugin",

  // 插件显示名称
  name: "Hello Plugin",

  // 插件描述
  description: "一个简单的示例插件，演示基本功能",

  // 配置 Schema，使用空 Schema
  configSchema: emptyPluginConfigSchema(),

  // 注册函数，接收 api 对象作为参数
  // api 对象提供了所有插件注册功能
  register(api: OpenClawPluginApi) {
    // 注册一个自定义命令 /hello
    // 这个命令不需要 AI 处理，直接返回固定文本
    api.registerCommand({
      // 命令名称（不带斜杠）
      name: "hello",

      // 命令描述，显示在帮助信息中
      description: "向用户问好",

      // 是否需要授权用户才能使用
      requireAuth: false,

      // 命令处理函数
      handler: (ctx) => {
        // 返回文本消息
        // ctx 包含命令上下文信息
        return {
          text: `你好！这是来自 Hello Plugin 的问候！\n当前渠道: ${ctx.channel}`,
        };
      },
    });

    // 记录日志，表示插件已加载
    api.logger.info("[hello-plugin] 插件已加载");
  },
};

// 导出插件对象（必须使用 default 导出）
export default helloPlugin;
```

## 2.3 插件结构

完成后的插件目录结构：

```
hello-plugin/
├── openclaw.plugin.json    # 插件清单（必需）
└── index.ts                # 插件入口（必需）
```

## 2.4 加载和测试插件

### 2.4.1 启用插件

```bash
# 列出所有已发现的插件
openclaw plugins list

# 启用插件
openclaw plugins enable hello-plugin

# 查看插件详情
openclaw plugins info hello-plugin
```

### 2.4.2 重启 Gateway

插件配置更改后需要重启 Gateway：

```bash
# 停止当前 Gateway（如果正在运行）
# 然后重新启动
openclaw gateway
```

### 2.4.3 测试插件

通过任意消息渠道发送 `/hello` 命令，您应该收到类似这样的回复：

```
你好！这是来自 Hello Plugin 的问候！
当前渠道: telegram
```

## 2.5 添加配置选项

让我们扩展插件，添加可配置的问候语。

### 2.5.1 更新清单文件

```json
{
  "id": "hello-plugin",
  "name": "Hello Plugin",
  "description": "一个简单的示例插件，演示基本功能",
  "version": "1.0.0",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "greeting": {
        "type": "string",
        "default": "你好！这是来自 Hello Plugin 的问候！",
        "description": "自定义问候语"
      },
      "showChannel": {
        "type": "boolean",
        "default": true,
        "description": "是否显示当前渠道信息"
      }
    }
  },
  "uiHints": {
    "greeting": {
      "label": "问候语",
      "placeholder": "输入自定义问候语..."
    },
    "showChannel": {
      "label": "显示渠道信息",
      "help": "开启后会在回复中显示消息来源渠道"
    }
  }
}
```

### 2.5.2 更新入口文件

```typescript
// 导入类型和工具函数
import type { OpenClawPluginApi } from "openclaw/plugin-sdk/core";

// 定义配置接口
interface HelloPluginConfig {
  greeting?: string;      // 问候语
  showChannel?: boolean;  // 是否显示渠道信息
}

// 定义插件对象
const helloPlugin = {
  id: "hello-plugin",
  name: "Hello Plugin",
  description: "一个简单的示例插件，演示基本功能",

  // 定义配置 Schema
  configSchema: {
    // 解析配置的函数
    parse(value: unknown): HelloPluginConfig {
      // 如果配置不存在或不是对象，返回空对象
      if (!value || typeof value !== "object" || Array.isArray(value)) {
        return {};
      }
      // 类型断言并返回
      return value as HelloPluginConfig;
    },

    // JSON Schema 定义
    jsonSchema: {
      type: "object",
      additionalProperties: false,
      properties: {
        greeting: {
          type: "string",
          default: "你好！这是来自 Hello Plugin 的问候！",
        },
        showChannel: {
          type: "boolean",
          default: true,
        },
      },
    },

    // UI 提示信息
    uiHints: {
      greeting: {
        label: "问候语",
        placeholder: "输入自定义问候语...",
      },
      showChannel: {
        label: "显示渠道信息",
        help: "开启后会在回复中显示消息来源渠道",
      },
    },
  },

  // 注册函数
  register(api: OpenClawPluginApi) {
    // 获取插件配置
    // api.pluginConfig 包含用户配置的值
    const config = this.configSchema.parse(api.pluginConfig) as HelloPluginConfig;

    // 解析配置值，设置默认值
    const greeting = config.greeting ?? "你好！这是来自 Hello Plugin 的问候！";
    const showChannel = config.showChannel ?? true;

    // 注册命令
    api.registerCommand({
      name: "hello",
      description: "向用户问好",
      requireAuth: false,

      handler: (ctx) => {
        // 构建回复消息
        let message = greeting;

        // 如果配置了显示渠道信息，添加到消息中
        if (showChannel) {
          message += `\n当前渠道: ${ctx.channel}`;
        }

        return { text: message };
      },
    });

    api.logger.info("[hello-plugin] 插件已加载");
  },
};

// 导出插件
export default helloPlugin;
```

### 2.5.3 配置插件

在 OpenClaw 配置文件中添加：

```json5
{
  plugins: {
    entries: {
      "hello-plugin": {
        enabled: true,
        config: {
          greeting: "欢迎来到 OpenClaw！",
          showChannel: true,
        },
      },
    },
  },
}
```

## 2.6 注册 AI 工具

让我们为 Agent 添加一个可调用的工具。

### 2.6.1 更新入口文件

```typescript
// 导入类型
import type { OpenClawPluginApi } from "openclaw/plugin-sdk/core";
import { Type } from "@sinclair/typebox";  // 用于定义工具参数 Schema

// 定义插件对象
const helloPlugin = {
  id: "hello-plugin",
  name: "Hello Plugin",
  description: "一个简单的示例插件，演示基本功能",

  configSchema: {
    parse: (value: unknown) => value || {},
    jsonSchema: { type: "object", additionalProperties: false },
  },

  register(api: OpenClawPluginApi) {
    // 注册 Agent 工具
    // Agent 可以在对话中调用这个工具
    api.registerTool({
      // 工具名称（使用 snake_case）
      name: "say_hello",

      // 工具显示标签
      label: "Say Hello",

      // 工具描述，Agent 会根据这个描述决定是否调用
      description: "向指定用户发送问候语。当需要友好地问候用户时使用此工具。",

      // 定义工具参数 Schema
      // 使用 Typebox 定义类型安全的 Schema
      parameters: Type.Object({
        // 用户名参数
        user_name: Type.String({
          description: "要问候的用户名",
        }),
        // 语言参数（可选）
        language: Type.Optional(
          Type.String({
            description: "问候语的语言，如 'zh'、'en'",
            default: "zh",
          }),
        ),
      }),

      // 工具执行函数
      // _toolCallId: 工具调用唯一标识
      // params: 解析后的参数
      async execute(_toolCallId, params) {
        // 获取参数值
        const userName = params?.user_name ?? "朋友";
        const language = params?.language ?? "zh";

        // 根据语言生成问候语
        let greeting: string;
        switch (language) {
          case "en":
            greeting = `Hello, ${userName}! Nice to meet you!`;
            break;
          case "zh":
          default:
            greeting = `你好，${userName}！很高兴见到你！`;
        }

        // 返回工具结果
        // 必须返回特定格式的内容
        return {
          content: [
            {
              type: "text" as const,
              text: greeting,
            },
          ],
        };
      },
    });

    // 同时保留命令注册
    api.registerCommand({
      name: "hello",
      description: "向用户问好",
      requireAuth: false,
      handler: (ctx) => ({
        text: `你好！当前渠道: ${ctx.channel}`,
      }),
    });

    api.logger.info("[hello-plugin] 插件已加载，包含 1 个工具和 1 个命令");
  },
};

export default helloPlugin;
```

### 2.6.2 测试工具

重启 Gateway 后，Agent 现在可以使用 `say_hello` 工具。在对话中尝试说：

```
请用英文向 Alice 问好
```

Agent 可能会调用工具并返回：

```
Hello, Alice! Nice to meet you!
```

## 2.7 开发工作流

### 2.7.1 推荐的开发流程

```
1. 修改插件代码
      ↓
2. 重启 Gateway（或使用热重载）
      ↓
3. 测试功能
      ↓
4. 检查日志
      ↓
5. 返回步骤 1（如有需要）
```

### 2.7.2 调试技巧

```bash
# 查看 Gateway 日志
tail -f ~/.openclaw/gateway.log

# 在 macOS 上使用脚本
./scripts/clawlog.sh -f

# 只查看插件相关日志
tail -f ~/.openclaw/gateway.log | grep plugin
```

### 2.7.3 常见问题排查

| 问题 | 解决方案 |
|------|----------|
| 插件未被发现 | 检查目录位置和 `openclaw.plugin.json` |
| 配置不生效 | 确保重启了 Gateway |
| 工具调用失败 | 检查参数 Schema 是否正确 |
| 类型错误 | 确保导入正确的类型定义 |
