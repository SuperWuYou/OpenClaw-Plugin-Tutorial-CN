# 第一章：介绍

## 1.1 什么是 OpenClaw 插件？

OpenClaw 插件是一种**小型代码模块**，用于扩展 OpenClaw 的核心功能。通过插件系统，您可以：

- 添加新的**消息渠道**（如 Discord、Matrix、Signal 等）
- 注册自定义的**AI 工具**供 Agent 调用
- 创建**后台服务**执行定时任务或长期运行的操作
- 扩展**HTTP 路由**提供 Webhook 或 API 端点
- 注册**CLI 命令**增强命令行功能
- 实现**提供商认证**支持新的 AI 模型提供商

## 1.2 为什么需要插件？

OpenClaw 采用**插件化架构**的设计理念：

```
┌─────────────────────────────────────────────────────┐
│                    OpenClaw Core                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌────────┐ │
│  │ Gateway │  │  Agent  │  │ Channel │  │ Config │ │
│  └────┬────┘  └────┬────┘  └────┬────┘  └───┬────┘ │
│       │            │            │            │      │
│       └────────────┴────────────┴────────────┘      │
│                          │                          │
│                    Plugin API                       │
└──────────────────────────┬──────────────────────────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
┌───┴───┐            ┌─────┴─────┐          ┌─────┴─────┐
│ 插件A │            │  插件B    │          │  插件C    │
│工具/服务│           │ 消息渠道  │          │ 提供商认证 │
└───────┘            └───────────┘          └───────────┘
```

### 插件的优势

| 优势 | 说明 |
|------|------|
| **模块化** | 功能独立封装，按需加载 |
| **可扩展** | 无需修改核心代码即可添加新功能 |
| **可维护** | 插件独立开发、测试和部署 |
| **安全性** | 通过 SDK 限制插件访问范围 |
| **配置驱动** | 完整的配置验证和 UI 支持 |

## 1.3 插件能做什么？

### 1.3.1 注册 Agent 工具

为 AI Agent 提供新的能力，例如：

```typescript
// 注册一个天气查询工具
api.registerTool({
  name: "get_weather",
  label: "Get Weather",
  description: "获取指定城市的天气信息",
  parameters: Type.Object({
    city: Type.String({ description: "城市名称" }),
  }),
  async execute(toolCallId, params) {
    const weather = await fetchWeather(params.city);
    return { content: [{ type: "text", text: weather }] };
  },
});
```

### 1.3.2 注册消息渠道

支持新的聊天平台，例如：

```typescript
// 注册一个新的消息渠道
api.registerChannel({
  plugin: {
    id: "my-channel",
    meta: { label: "My Channel" },
    capabilities: { chatTypes: ["direct", "group"] },
    outbound: {
      deliveryMode: "direct",
      sendText: async (ctx) => ({
        channel: "my-channel",
        messageId: "msg-123",
      }),
    },
  },
});
```

### 1.3.3 注册生命周期钩子

在关键时刻执行代码，例如：

```typescript
// 在消息发送前进行处理
api.on("message_sending", async (event, ctx) => {
  console.log(`发送消息到: ${event.to}`);
  return { content: event.content.toUpperCase() };
});
```

### 1.3.4 注册 HTTP 路由

提供 Webhook 或 API 端点，例如：

```typescript
// 注册一个 HTTP 路由
api.registerHttpRoute({
  path: "/plugins/my-plugin/webhook",
  auth: "plugin",
  handler: async (req, res) => {
    res.writeHead(200);
    res.end("OK");
  },
});
```

### 1.3.5 注册 CLI 命令

扩展命令行功能，例如：

```typescript
// 注册一个自定义 CLI 命令
api.registerCli(({ program }) => {
  program
    .command("mycommand")
    .description("执行自定义命令")
    .action(() => {
      console.log("命令执行成功！");
    });
});
```

### 1.3.6 注册后台服务

长期运行的后台任务，例如：

```typescript
// 注册一个后台服务
api.registerService({
  id: "my-service",
  start: async () => {
    console.log("服务启动");
  },
  stop: async () => {
    console.log("服务停止");
  },
});
```

## 1.4 插件系统架构

### 1.4.1 核心组件

```
src/
├── plugins/              # 插件核心系统
│   ├── types.ts         # 类型定义
│   ├── loader.ts        # 插件加载器
│   ├── config-schema.ts # 配置 Schema 工具
│   └── runtime/         # 插件运行时
├── plugin-sdk/          # 插件 SDK
│   ├── core.ts          # 核心 API
│   ├── telegram.ts      # Telegram SDK
│   ├── discord.ts       # Discord SDK
│   └── ...              # 其他渠道 SDK
└── extensions/          # 内置插件目录
    ├── voice-call/      # 语音通话插件
    ├── memory-core/     # 内存搜索插件
    └── ...              # 其他插件
```

### 1.4.2 插件生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                      插件生命周期                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 发现阶段                                                 │
│     ├── 扫描插件目录                                         │
│     ├── 读取 openclaw.plugin.json                           │
│     └── 验证配置 Schema                                      │
│                                                             │
│  2. 加载阶段                                                 │
│     ├── 安全检查（路径遍历、权限等）                          │
│     ├── 动态导入插件模块                                      │
│     └── 创建插件 API 实例                                    │
│                                                             │
│  3. 注册阶段                                                 │
│     ├── 调用 register(api)                                  │
│     ├── 注册工具、钩子、服务等                                │
│     └── 验证注册内容                                         │
│                                                             │
│  4. 运行阶段                                                 │
│     ├── 处理请求和事件                                       │
│     └── 执行插件逻辑                                         │
│                                                             │
│  5. 停止阶段                                                 │
│     └── 调用服务的 stop() 方法                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 1.5 插件类型概览

| 类型 | 说明 | 示例 |
|------|------|------|
| **工具插件** | 为 Agent 提供新工具 | `diffs`、`llm-task` |
| **渠道插件** | 支持新的消息平台 | `discord`、`matrix`、`telegram` |
| **服务插件** | 后台服务 | `voice-call`、`diagnostics-otel` |
| **提供商插件** | AI 模型提供商认证 | `google-gemini-cli-auth` |
| **内存插件** | 内存/记忆功能 | `memory-core`、`memory-lancedb` |
